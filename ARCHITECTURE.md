# Architecture

This project uses Twilio Programmable Voice to place outbound calls from one verified Twilio number to the Pretty Good AI assessment number. A Flask app exposes TwiML webhooks: `/voice/start` waits for the practice agent's greeting, `/voice/reply` receives Twilio speech-recognition results for the agent's turn, generates the next patient response, speaks it with Twilio text-to-speech, then gathers the next agent turn. Each call is locked to a scenario ID, and every agent/patient turn is appended to both JSONL and plain-text transcript files.

The patient simulator uses a lightweight LLM decision loop instead of a rigid script so it can adapt to ambiguous agent responses while still steering toward a defined test objective. The scenarios intentionally cover happy paths, operational questions, safety-sensitive refill behavior, ambiguity, and edge cases like weekend scheduling and interruptions. Twilio's call recording callback downloads MP3 audio, while export scripts package transcripts, recordings, and a call index into the `deliverables/` folder. The system is intentionally simple: local Flask plus ngrok is enough to ship working calls quickly, while clear guardrails prevent accidental calls to any number except `+1-805-439-8008`.
